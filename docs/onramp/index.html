Convert Fiat to Crypto

Magic Bank Account​

Once a Customer has completed onboarding, Harbour will generate them a Magic Bank Account. This is an account issued in the Customer’s name, which is linked to their whitelisted addresses.

When a payment is received on the customer’s Magic Bank Account, Harbour automatically converts the received fiat currency into a supported crypto asset and transfers on a supported blockchain network to the customer's whitelisted address.

Instructions as to which supported crypto asset to send on which supported blockchain network and to which whitelisted wallet are to provided to Harbour by inclusion of a simple Payment Reference in the bank transfer to the Magic Bank Account.

Displaying the Magic Account​

A single API call (GetAccountInfo) allows client apps to fetch all the information required to display the magic bank account’s coordinates to the customer, along with the list of supported assets and daily/transaction limits.

Whitelisting Wallets​

Customers can only receive supported crypto assets on blockchain addresses that have been notified to Harbour for that purpose.

The GetAccountInfo also returns the list of whitelisted wallets (addresses). Adding a new address is as easy as invoking the WhitelistAddress API with a signature proving ownership of the address.

Payment References​

The inclusion of a payment reference in a bank transfer to a Magic Bank Account tells Harbour:

  • the supported crypto asset to send to the recipient
  • the supported blockchain on which to execute the transfer
  • the whitelisted address to send to

Harbour payment references follow a simple intuitive structure, designed to be memorable and easy to use: {Chain}{Asset Ticker}{Customer whitelisted address}. E.g. To receive USDC on the Ethereum blockchain to my first whitelisted address, my payment reference is: EUSDC1. The number increases incrementally each time a Customer whitelists a new address.

Our payment reference parsing logic is able to recognise ambiguous characters, such as a one (1) mistyped as lower case L (l) to ensure maximum success rate.

warning

Note: it should be made very clear to Customers in the user interface that a valid payment reference is required for all fiat to crypto asset conversions. Transfers received on a Magic Bank Account without a valid payment reference will be bounced back to the originating bank account.

Supported Payees​

We currently restrict fiat payments exclusively from bank accounts in the customer's name. Our next product iteration will include support for third-party payments. In the meanwhile, a mismatch between the payer and payee names will result in the payment being bounced back.
